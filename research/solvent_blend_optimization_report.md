# HSPによる溶剤ブレンド最適化 調査レポート

調査日: 2026-03-15

---

## 1. エグゼクティブサマリー

Hansen溶解度パラメータ（HSP）を利用した溶剤ブレンド最適化手法を調査した。HSPの混合則（体積加重線形平均）により、複数溶剤の混合物のHSP値を予測でき、単独溶剤では到達できないHSP空間上の任意の点を混合により実現できる。本調査ではグリッドサーチ、シンプレックス法、遺伝的アルゴリズムの3手法を比較し、2-3成分系での最適化アルゴリズムとして網羅的グリッドサーチを選定した。

**結論:** 体積分率の線形混合則はHansen（2007）により理論的・実験的に支持されており、2-3成分系のグリッドサーチは計算量が許容範囲内で、最適解の見落としがない確実な手法である。

---

## 2. 理論的背景

### 2.1 HSPの混合則

HSPは溶解度パラメータの定義から、体積加重線形平均（linear volume-weighted average）で混合物のHSP値を算出できる。n成分の混合物について：

```
δD_mix = Σ φi · δDi
δP_mix = Σ φi · δPi
δH_mix = Σ φi · δHi
```

ここで：
- `φi` = 成分iの体積分率（Σφi = 1）
- `δDi, δPi, δHi` = 成分iのHSP値（MPa^1/2）

### 2.2 線形混合則の理論的根拠

溶解度パラメータδは凝集エネルギー密度の平方根として定義される：

```
δ² = E_coh / V_m
```

ここで `E_coh` は凝集エネルギー、`V_m` はモル体積である。

混合物の凝集エネルギー密度について、理想混合を仮定すると：

```
E_coh,mix / V_mix = Σ (φi · E_coh,i / V_i) = Σ (φi · δi²)
```

これは厳密にはδ²が線形混合であることを意味するが、HSPの各成分（δD, δP, δH）についてはHansen（2007）が体積分率に対する線形混合を推奨しており、以下の理由による：

1. **実験的検証**: 多くの系で線形混合が実験値と良好に一致
2. **実用的近似**: δ²の線形混合とδの線形混合の差は、通常の混合比範囲で小さい
3. **直感的**: HSP空間上で2成分の混合は2点を結ぶ直線上を移動

### 2.3 HSP空間における混合の幾何学的意味

2成分混合の場合、混合物のHSP値は3次元HSP空間において2つの純溶剤の点を結ぶ**線分上**に位置する。体積分率φを0→1に変化させると、溶剤Aの点から溶剤Bの点へ直線的に移動する。

3成分混合の場合、3つの純溶剤の点を頂点とする**三角形の内部**のすべての点が到達可能となる。

```
2成分: 線分（1次元の自由度）
3成分: 三角形面（2次元の自由度）
4成分: 四面体体積（3次元の自由度）
```

この幾何学的性質により、**適切な溶剤の組合せを選べばHSP空間上の広い範囲をカバー**できる。

## 3. 最適化アルゴリズムの比較

### 3.1 問題定式化

溶剤ブレンド最適化は以下の制約付き最適化問題である：

```
目的: min Ra(δ_mix, δ_target) = √[4·(δD_mix - δD_t)² + (δP_mix - δP_t)² + (δH_mix - δH_t)²]
制約: Σφi = 1, φi ≥ 0 (全てのiについて)
```

ここで `δ_target` は目標HSP値（例: 対象ポリマーのHSP中心）。

### 3.2 グリッドサーチ（網羅的探索） — **採用**

体積分率の全組合せを一定刻み幅（stepSize）で列挙し、各組合せについてRaを計算、最小Raの組合せを選択する。

**2成分系のアルゴリズム:**

```
for φ1 = 0 to 1 step s:
    φ2 = 1 - φ1
    δ_mix = φ1·δ1 + φ2·δ2
    Ra = distance(δ_mix, δ_target)
    if Ra < best: update best
```

**3成分系のアルゴリズム:**

```
for φ1 = 0 to 1 step s:
    for φ2 = 0 to (1-φ1) step s:
        φ3 = 1 - φ1 - φ2
        δ_mix = φ1·δ1 + φ2·δ2 + φ3·δ3
        Ra = distance(δ_mix, δ_target)
        if Ra < best: update best
```

**計算量:**

| 成分数 | 組合せ数 | stepSize=0.05 | stepSize=0.01 |
|--------|---------|--------------|--------------|
| 2 | 1/s + 1 | 21 | 101 |
| 3 | (1/s+1)(1/s+2)/2 | 231 | 5,151 |

溶剤ペア数も考慮した全体の計算量：

```
2成分: O(N² · S)     N=溶剤数, S=1/stepSize
3成分: O(N³ · S²)
```

N=50溶剤、stepSize=0.05の場合：
- 2成分: 50²×20 = 50,000回
- 3成分: 50³×20² = 50,000,000回

3成分系では計算量が大きくなるが、現代のマシンでは数秒以内に処理可能。

### 3.3 シンプレックス法（Nelder-Mead）

多変数の非線形最適化手法。初期点から反射・膨張・収縮操作で最適解を探索する。

**利点:**
- 勾配計算不要
- 高次元でも適用可能
- 収束が速い

**欠点:**
- **局所最適解に陥る可能性**
- 初期点依存性が高い
- 制約条件（φi≥0, Σφi=1）の扱いが煩雑
- 溶剤の組合せ選択は離散問題であり、連続最適化手法では直接扱えない

### 3.4 遺伝的アルゴリズム（GA）

生物の進化過程を模倣したメタヒューリスティクス手法。

**利点:**
- 大域的最適解を見つけやすい
- 離散・連続の混合問題に適用可能
- 並列化が容易

**欠点:**
- **パラメータチューニング（集団サイズ、突然変異率等）が必要**
- **再現性が低い**（確率的手法のため実行ごとに異なる結果）
- 2-3成分程度の小規模問題にはオーバースペック
- 実装が複雑

### 3.5 手法比較まとめ

| 評価項目 | グリッドサーチ | シンプレックス | 遺伝的アルゴリズム |
|---------|-------------|-------------|----------------|
| 最適解保証 | **確実**（全探索） | 局所最適のみ | 準最適（確率的） |
| 再現性 | **完全に再現** | 初期点依存 | 実行ごとに異なる |
| 実装容易性 | **非常に容易** | 中程度 | 複雑 |
| 2成分計算速度 | 非常に速い | 速い | 遅い |
| 3成分計算速度 | 速い（許容範囲） | 速い | 中程度 |
| 4+成分 | 指数的に増大 | 適用可能 | 適用可能 |
| 制約条件処理 | 自然に満たす | 追加処理必要 | ペナルティ関数等 |

**グリッドサーチ採用の理由:**

1. **網羅性**: 全組合せを評価するため、最適解を見逃さない
2. **決定論的**: 同じ入力に対して常に同じ結果（再現性が重要）
3. **2-3成分で十分**: 実用上、4成分以上のブレンドは稀
4. **TopN提示**: 上位N件の候補を容易に抽出できる（局所最適法では困難）
5. **実装の簡潔さ**: バグが混入しにくく、検証が容易

## 4. 実装における設計判断

### 4.1 topN トランケーション

全組合せを計算後、Ra値でソートして上位N件（デフォルト: topN=10）のみを返す。

**理由:**
- 3成分系では数百万の組合せが生成される
- 全結果をメモリに保持してUIに渡すのは非効率
- 実用上、ユーザーが必要とするのは上位数件の最適候補のみ

**実装方針:**
- 計算中はMin-Heapまたは固定長の優先度付きキューを使用
- メモリ使用量をO(topN)に抑制
- 同一Ra値の場合は成分数が少ない方を優先（シンプルな配合を推奨）

### 4.2 ステップサイズの選択

| stepSize | 精度 | 2成分の組合せ数 | 3成分の組合せ数 |
|----------|------|----------------|----------------|
| 0.10 | 低 | 11 | 66 |
| 0.05 | 中 | 21 | 231 |
| 0.02 | 高 | 51 | 1,326 |
| 0.01 | 最高 | 101 | 5,151 |

デフォルトはstepSize=0.05（5%刻み）を推奨。実用上、±2.5%以内の体積分率誤差は調合精度の範囲内であり、計算速度と精度のバランスが良い。

### 4.3 溶剤フィルタリング

最適化対象の溶剤プールには以下のフィルタリングが有効：

1. **コスト制約**: 高価な溶剤を除外
2. **安全性制約**: 特定の危険等級（GHS分類）を除外
3. **沸点制約**: 蒸発速度の調整のため沸点範囲を指定
4. **相互溶解性**: 混合する溶剤同士が相溶であることの確認

### 4.4 結果の表示

最適化結果には以下を含める：

```
組合せランク: 1
溶剤A: トルエン (φ=0.60)
溶剤B: メタノール (φ=0.40)
混合HSP: δD=17.1, δP=6.2, δH=9.3
ターゲットとのRa: 1.23
RED: 0.15
```

## 5. 非理想混合と限界事項

### 5.1 体積変化（過剰体積）

理想混合では混合体積は各成分の体積の和に等しい：

```
V_mix,ideal = Σ ni · V_m,i
```

実際には過剰体積 `V^E` が生じる：

```
V_mix,real = V_mix,ideal + V^E
```

| 系 | 過剰体積の目安 | 影響度 |
|---|------------|-------|
| 非極性-非極性 | 小さい（< 0.5%） | 無視可能 |
| 極性-非極性 | 中程度（0.5-2%） | 軽微 |
| 水-有機溶媒 | 大きい（> 2%） | 注意が必要 |
| 強い水素結合系 | 非常に大きい | 線形混合則の適用に注意 |

エタノール-水系は過剰体積が特に大きい代表例で、体積収縮が約3.5%に達する。このような系ではHSPの線形混合則からの乖離が大きくなる。

### 5.2 温度効果

HSP値は温度依存性を持つが、混合則自体は温度に依存しない。ただし：

- 高温で溶媒の蒸発速度差が大きい場合、経時的に混合比が変化
- 溶解度パラメータの温度係数はδDが最大（分散力の温度依存性）
- 通常25°Cのデータベース値を使用するため、高温・低温条件では補正が必要

### 5.3 その他の限界

1. **速度論的効果**: HSPは熱力学的親和性のみを予測し、溶解速度や拡散速度は考慮しない
2. **3成分を超えるブレンド**: 計算量の爆発的増大（4成分: O(N⁴·S³)）により実用的でない
3. **特殊な相互作用**: 反応性溶剤（エポキシ硬化剤等）、共沸混合物、特殊な分子間相互作用はHSPモデルの範囲外
4. **蒸発後の膜物性**: コーティング用途では溶剤蒸発後の膜品質（光沢、レベリング、ピンホール）はHSPでは予測不可

## 6. 応用分野

### 6.1 塗料・コーティング配合

最も一般的な応用。ポリマー樹脂（バインダー）のHSP値をターゲットとし、溶剤混合物のHSPを樹脂のHansen球内に配置する。

**実例: エポキシ樹脂用溶剤ブレンド**

エポキシ樹脂の典型的HSP: δD=20.0, δP=12.0, δH=11.5

| 溶剤 | δD | δP | δH | 単独Ra |
|------|----|----|-----|--------|
| キシレン | 17.8 | 1.0 | 3.1 | 13.6 |
| MEK | 16.0 | 9.0 | 5.1 | 10.3 |
| ブタノール | 16.0 | 5.7 | 15.8 | 10.7 |

最適ブレンド例: キシレン55% + MEK30% + ブタノール15%

```
δD_mix = 0.55×17.8 + 0.30×16.0 + 0.15×16.0 = 17.0
δP_mix = 0.55×1.0 + 0.30×9.0 + 0.15×5.7 = 4.1
δH_mix = 0.55×3.1 + 0.30×5.1 + 0.15×15.8 = 5.6
Ra = √[4×(20.0-17.0)² + (12.0-4.1)² + (11.5-5.6)²] = √[36 + 62.4 + 34.8] = 11.5
```

各溶剤単独よりは改善するが、さらなる最適化にはグリッドサーチが有効。

### 6.2 医薬品共溶媒

難溶性APIの溶解性向上のため、水と有機溶媒の混合比を最適化する。

### 6.3 洗浄溶剤

環境規制（VOC削減）の下で、被洗浄物に適した低毒性溶剤ブレンドを設計する。

## 7. 計算例：2成分最適化の詳細

### ターゲット: ポリカーボネート

ポリカーボネートHSP: δD=20.0, δP=10.0, δH=6.0, R₀=10.0

### 候補溶剤（3種から2種を選択）

| 溶剤 | δD | δP | δH |
|------|----|----|-----|
| アセトン | 15.5 | 10.4 | 7.0 |
| トルエン | 18.0 | 1.4 | 2.0 |
| THF | 16.8 | 5.7 | 8.0 |

### 組合せ1: アセトン + トルエン（stepSize=0.10）

| φ_アセトン | φ_トルエン | δD_mix | δP_mix | δH_mix | Ra |
|-----------|-----------|--------|--------|--------|------|
| 0.0 | 1.0 | 18.0 | 1.4 | 2.0 | 10.0 |
| 0.1 | 0.9 | 17.8 | 2.3 | 2.5 | 9.1 |
| 0.2 | 0.8 | 17.5 | 3.2 | 3.0 | 8.2 |
| 0.3 | 0.7 | 17.3 | 4.1 | 3.5 | 7.4 |
| 0.4 | 0.6 | 17.0 | 5.0 | 4.0 | 6.8 |
| 0.5 | 0.5 | 16.8 | 5.9 | 4.5 | 6.5 |
| 0.6 | 0.4 | 16.5 | 6.8 | 5.0 | 6.6 |
| 0.7 | 0.3 | 16.3 | 7.7 | 5.5 | 7.1 |
| 0.8 | 0.2 | 16.0 | 8.6 | 6.0 | 8.0 |
| 0.9 | 0.1 | 15.8 | 9.5 | 6.5 | 8.5 |
| 1.0 | 0.0 | 15.5 | 10.4 | 7.0 | 9.1 |

**最適比**: アセトン50% + トルエン50%（Ra=6.5）

### 組合せ2: THF単独

THF: Ra = √[4×(20.0-16.8)² + (10.0-5.7)² + (6.0-8.0)²] = √[40.96 + 18.49 + 4.0] = 7.96

THF単独よりアセトン+トルエンのブレンドの方がRaが小さく、ブレンドの優位性が示される。

## 9. 実装判断のまとめ

| 判断項目 | 決定 | 理由 |
|---------|------|------|
| 混合則 | 体積分率線形平均 | Hansen（2007）推奨、実験的に検証済み |
| 最適化手法 | グリッドサーチ | 網羅的・決定論的・2-3成分で計算量許容範囲 |
| デフォルトstepSize | 0.05 | 精度と速度のバランス |
| 最大成分数 | 3 | 4成分以上は計算量・実用性の観点から非推奨 |
| topN | 10 | メモリ効率とUI表示の実用性 |
| 溶剤フィルタリング | 対応 | 安全性・コスト等の実務的制約への対応 |
| 体積収縮補正 | 非対応 | 多くの系で影響は軽微、データ不足 |
